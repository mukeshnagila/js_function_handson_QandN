Q.5 - 
                "Print output: 
                    var x = 21;
                    girl ();
                    console.log(x)
                    function girl() {
                        console.log(x);
                        var x = 20;
                    };" .........?


Ans - 
       It's prints "undefined" And after that "21"..


Explain--


In this code, a global variable x is declared and assigned the value of "21". Then, the girl() function is called. Inside the function, a local variable "x" is declared, overshadowing the global "x". The first "console.log(x)" statement inside the function tries to access the local "x", but since it is declared but not assigned a value yet, its value is "undefined". After the function call, the "console.log(x)" statement outside the function logs the value of the global "x", which is "21". Therefore, the output is "undefined" And "21"...

Q.7 - Write a function that accepts parameter n and returns factorial of n


Ans - 
       
                    def factorial(n):
                        if n == 0 or n == 1:
                            return 1
                        else:
                    return n * factorial(n-1)


Explain--


The factorial of a number 'n' is the product of all positive integers from '1' to 'n'. In the given function, we use recursion to calculate the factorial. If n is 0 or '1', the factorial is defined as '1'. Otherwise, we multiply 'n' with the factorial of 'n-1'. This process continues recursively until n becomes '0' or '1', and then the final factorial value is returned...



Q.8 - 
                "Guess the Output

                    function FindSum(a, b){
                        return a + b;
                    }

                    function DisplayData(data, batch){
                        console.log(`i am from ${data} and My batch is EA${batch}`)
                    }

                    DisplayData(""PrepBytes"", FindSum(10, 9));" .........?


Ans - 
          i am from PrepBytes and My batch is EA19.


Explain--


The code defines two functions: "FindSum" and "DisplayData". The "FindSum" function takes two parameters 'a' and 'b' and returns their 'sum'. The "DisplayData" function takes two parameters data and batch, and it uses template literals to log a string to the console.
In the given code, "DisplayData" is called with two arguments: ""PrepBytes"" and the result of FindSum(10, 9).
"FindSum(10, 9)" evaluates to '19'.
The template literal in "DisplayData" uses the values "PrepBytes" and "19" to form the string "i am from PrepBytes and My batch is EA19".
This string is then logged to the console. Hence, the output of the code is "i am from PrepBytes and My batch is EA19"...

Q.10 - 
                "Guess the output

                    var a = 10;
                    (function (){
                        console.log(a);
                        var a = 20;
                    })();".........?


Ans - 
      It's prints undefined.


Explain--


The code declares a variable a and assigns it a value of "10". Then, it defines an immediately invoked function expression (IIFE). Inside the IIFE, a new variable "a" is declared and assigned a value of "20". However, due to JavaScript's hoisting behavior, the declaration of the local variable a is hoisted to the top of the function scope, but the assignment '(var a = 20)' is not hoisted. Therefore, when "console.log(a)" is executed, the local variable a exists but hasn't been assigned a value yet. As a result, the value of a is "undefined" at that point ...
